# 补充实验方案（报告可直接采用）

## 1. 对照与样本

对每个模型固定版本、端点和日期，温度设为 0；若服务端不支持确定性采样，在日志中记录实际参数。GAOKAO-Bench 客观题按学科分层抽样，优先全量；TruthfulQA 采用完整 817 题或在资源受限时分层随机抽取不少于 100 题。所有随机种子、提示词与数据版本应写入运行目录。

比较三种同成本边界清楚的方法：直接回答（Direct）、质疑后重答（Challenge）和 Chain-of-Verification（CoVe）。Challenge 只对初答错误或低置信样本二次调用；CoVe 严格采用“草稿—验证问题—独立回答验证问题—重写”四步，不把草稿直接带入验证答案。

所有开放题方法都使用同一系统提示词，要求回答不超过 220 个英文词（或其他语言中的等价简洁长度），并将输出上限固定为 768 tokens。任何回答恰好达到上限的运行视为截断，不进入标注、阈值选择或最终图表。

截断排除按机械规则执行：`completion_tokens >= max_tokens` 的题目从分析集移除，并在 `runs/*_excluded.json` 公开题号和原因；原始逐题日志仍然保留。报告同时给出原始样本量与分析样本量，且对所有方法执行相同规则。

### FactBench 的固定划分与标注

先执行 `python -m src.fetch_factbench --out data/factbench/development_60.jsonl --n 60 --seed 20260812`。开发集仅用于选择 ConSC-Verify 风险阈值；阈值确定后，执行 `--n 200 --seed 20260813` 生成独立测试集。记录下载日期、数据集 URL、随机种子和文件 SHA-256。开放题回答以事实原子为单位，使用官方 VERIFY 的 `supported` / `unsupported` / `undecidable` 三类定义；若受计算资源限制改用人工复核，则两名评阅者独立标注、计算 Cohen's kappa，并保留分歧裁决表。

本实验实际观察到 DeepSeek V4 Flash 的返回 logprob 在开发运行中全部为 0，候选 token 为接口哨兵值，因此不将它作为概率置信度。首版“自评正确性”审计仅检出 4 条 AI 辅助初审错误中的 1 条，未达到预设的 80% 召回目标，作为失败消融保留。最终 ConSC-Verify 定义为 **Claim-Sensitive Calibrated Verification**：固定审计提示词根据回答的可核验断言面（日期、数字、实体、时效性及医疗/法律/金融/联络信息）输出 1–5 风险，而非让模型判断自身是否正确。仅以开发集上 `unsupported` 标签校准该分数阈值；测试集不再修改提示词、评分规则或阈值。

## 2. 指标

客观题主指标为准确率，另报告无法解析率、格式错误率和按学科的 Wilson 95% 置信区间。开放式事实问答报告完全正确、部分正确、错误/误导三类比例；两名评阅者独立标注，分歧由预先定义的规则处理，并报告 Cohen's kappa。缓解方法还报告相对基线的准确率差、平均调用次数、平均 token 和延迟。

## 3. 错误类型

每个错误最多标注一个主因：格式/指令跟随、事实知识、推理计算、实体或数值混淆、拒答不当、证据不足但过度自信。对“模型知道但首次未答对”的判断，只有在独立验证或外部证据支持时才可使用，不能仅凭模型自我声明。

## 4. 停止条件与呈现

先完成固定样本的一次无人工干预运行，再进行盲态人工复核。不得根据结果临时删除题目或改变解析规则；若修正规则，保留旧版并整体重跑。图表必须标注样本量、模型版本和运行日期；将逐题 JSONL、汇总表和绘图脚本一并公开。
